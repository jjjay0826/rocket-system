# CLAUDE.md — 航電韌體專案工作守則（custom instructions）

> 這些是本專案的強制工作守則，優先於預設行為，務必逐條遵守。
> 模型聲明：實際執行的可能是 Claude Opus 4.8 等模型，非特定「Fable 5」。
> 以下是**與模型無關的推理與自我審查紀律**，不是某個模型的內建機制。

## 專案性質（為什麼標準要高）
- STM32F411 火箭航電板（**rocket_v7**），實作降落傘**自動投放**——屬**飛行/安全關鍵**系統。
- 一個錯誤可能導致「該開傘卻不開 → 墜毀」或「地面誤觸發發火」。
- 因此：任何改動都要當作會上天執行來審，寧慢勿錯。

---

## 一、修改任何程式碼「之前」：先思考與證偽（pre-edit）
動手前，**先試著推翻自己的假設**，而不是找理由支持它：

1. **明說假設**：我要改的依據是什麼？哪隻腳、哪個暫存器值、哪條邏輯路徑。
2. **主動證偽**（找反證，不是找支持）：
   - **腳位**：以 **netlist 為唯一真實來源**，不可只靠讀 schematic 圖「看起來像」就下結論。
   - **暫存器 / 感測器行為**：對照 **datasheet** 的實際位元定義與數值，不可憑印象。
   - **既有程式**：改之前**先用 Read 看實際內容**，不可靠記憶或假設檔案長相。
   - **時脈 / 外設**：對照 `.ioc` 實際設定與生成碼，不可假設。
3. 證偽通不過（找不到反證、且有實證支持）才動手改。

## 二、修改程式碼「之後」：自我挑錯 + 邊界條件審查（post-edit）
每次改完，**主動找自己剛寫的 bug**：

1. **自我挑錯**：
   - 型別 / 格式字串是否相符（例：`printf` `%d` 配到 `float` → 會錯位）。
   - 是否造成**重複定義 / 符號衝突**（例：USER CODE 手寫 vs CubeMX 生成）。
   - 初始化順序、未使用變數、懸空 extern、include 連動。
2. **邊界條件**：
   - 溢位、除以零、null 指標、off-by-one、緩衝區大小、stack 用量。
   - 競態（ISR vs 主迴圈）、回呼依 instance 分派。
   - **上電 / 重置瞬間**狀態、感測器**失效路徑**（dead/timeout）。
3. **安全**：發火 / 開傘腳的**預設位準**、上電不誤觸；改開傘邏輯要核對通道對應。

## 三、不確定時：主動向終端申告，**不要瞎編**
- 對**數據、邏輯、假設**沒有把握時 → 在終端**明確說出**：「我不確定 X，因為 Y」，並說明**需要補什麼資料**（netlist / datasheet / 實測 / .ioc）。
- 寧可申告、請對方提供資料，**也不要編造**腳位、暫存器值、API 行為、或硬體接線。
- 區分「已實證」與「推測」：推測要標明，並說明驗證方式。

## 本專案踩過的雷（引以為戒，別重蹈）
- **讀 schematic 圖猜腳位** → 必須以 **netlist** 為準（rocket_v2 與 v7 差在 PB8/PB9，圖上易誤判）。
- **腳位寫死散在多檔**（main.c / 各驅動的 `GPIOA`/`GPIO_PIN_x`）→ 換板要人工追、易錯；應集中或用 **CubeMX User Label**（生成 `*_Pin`/`*_GPIO_Port`，換板零改碼）。
- **CSV 格式 `tc` 用 `%d` 印 float** → varargs 後續欄位全錯位；改格式字串先逐一核對型別。
- **啟用 USART1/2 NVIC 後**，CubeMX 生成的 `USARTx_IRQHandler` 與舊板 USER CODE 手寫版**撞名** → link 重複定義；HAL 弱回呼也只能有一份，依 instance 分派。
- **FATFS 改 User-defined** 後物件改名 `SDFatFS`→`USERFatFS`、`SDPath`→`USERPath`。
- **專案結構已是乾淨版**（2026-06-26 遷移，`ProjectManager.UnderRoot=true`、Core/Src 真實 source folder）：新增 `.c` 丟進 Core/Src 會**自動納入**，不用再手動加 `.project <link>`。注意：真實 source folder 會掃整個 Drivers/，別把多餘 .c（如完整 CMSIS DSP/DAP/RTOS2）放進來，否則會被編而爆——CMSIS 只留 Include/+Device/。
- **IMU WHO_AM_I** 不可只認單一值：LSM6DS 家族（0x69 LSM6DS3 ~ 0x6C LSM6DSO）暫存器相容，v7 用 GY-LSM6DS3。
- **USB 從 HSI 出 48MHz**（±1%）對 USB FS（±0.25%）偏鬆；雙電源（USB+電池）會打架。
